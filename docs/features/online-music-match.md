# 在线音乐匹配功能

## 功能概述

TPlayer 现已支持在线匹配歌曲元数据，包括歌词和封面图片。通过集成 QQ 音乐和网易云音乐 API，用户可以轻松为本地歌曲获取准确的元数据。

## 主要特性

### 1. 在线歌曲搜索
- 支持 QQ 音乐和网易云音乐两个音源
- 智能合并搜索结果，自动去重
- 显示歌曲封面、标题、艺术家、专辑等信息

### 2. 歌词获取
- 支持普通歌词（LRC）和滚动歌词（YRC）
- 自动缓存已获取的歌词，减少网络请求
- 支持歌词预览功能

### 3. 封面获取
- 支持多种尺寸封面（S/M/L/XL）
- 自动下载并缓存封面图片
- 支持封面预览功能

### 4. 智能匹配
- 使用字符串相似度算法（Levenshtein 距离）
- 自动匹配歌曲标题和艺术家
- 支持从文件名智能提取元数据

## 使用方法

### 通过编辑标签对话框

1. 在播放列表中选择歌曲，右键点击"编辑标签"
2. 点击"在线匹配标签"按钮
3. 在弹出的对话框中搜索歌曲
4. 从搜索结果中选择正确的歌曲
5. 点击"获取歌词"或"获取封面"预览
6. 点击"应用全部"将元数据应用到歌曲

### 自动匹配

1. 在编辑标签对话框中点击"从文件名匹配"
2. 系统会自动从文件名提取标题和艺术家
3. 自动搜索并匹配在线元数据

## 技术实现

### 前端组件

- `OnlineMatchModal.vue` - 在线匹配对话框组件
- `onlineMusicService.ts` - 在线音乐服务
- `cacheService.ts` - 缓存管理服务
- `stringSimilarity.ts` - 字符串相似度工具

### 后端命令

- `save_to_cache` - 保存文件到缓存
- `get_cached_file` - 从缓存获取文件
- `get_cache_dir` - 获取缓存目录
- `clear_cache` - 清除缓存

### 缓存策略

- 歌词缓存：localStorage 索引 + 文件系统存储
- 封面缓存：localStorage 索引 + 文件系统存储
- 缓存位置：系统应用数据目录

## API 端点

### QQ 音乐
- 搜索：`https://api.lxmusic.top/qq/search/{keyword}`
- 歌词：`https://api.lxmusic.top/qq/lyric/{songId}`
- 封面：`https://y.gtimg.cn/music/photo_new/T002R{s}x{s}M000{albumMid}.jpg`

### 网易云音乐
- 搜索：`https://music.163.com/api/search/get/web`
- 歌词：`https://music.163.com/api/song/lyric`
- 封面：从搜索结果直接获取

## 注意事项

1. 需要网络连接才能使用在线匹配功能
2. 缓存文件会占用磁盘空间，可定期清理
3. 部分歌曲可能无法找到匹配的元数据
4. API 可能有请求频率限制
